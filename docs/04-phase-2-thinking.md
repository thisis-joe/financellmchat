# 04. 2차 사고 흐름

2차의 핵심은 주제를 좁히는 것이었다.  
1차는 일반 금융 문서를 등록하고 검색하는 구조에 가까웠다. 하지만 실제 학습과 시연에는 더 명확한 도메인이 필요했다. 그래서 프로젝트 주제를 `BNK부산은행 상품공시 > 예금상품 > 적립식예금` PDF 기반 RAG로 고정했다.

이 판단에는 이유가 있었다. 부산은행 상품공시는 예금, 대출, 기타상품, 수수료 등 범위가 넓다. 처음부터 전체를 다루면 수집, 분류, 추출, 검색 품질 문제를 한꺼번에 만나게 된다. 그래서 사용자가 제공한 적립식예금 PDF부터 다루기로 했다. 범위를 좁히면 검색 품질 문제를 더 정확히 볼 수 있고, 나중에 다른 상품군을 같은 방식으로 확장할 수 있다.

먼저 `FinancialDocument`에 상품설명서에 필요한 메타데이터를 추가했다. 일반 문서라면 제목과 본문만으로도 검색이 가능하지만, 상품공시 RAG에서는 상품명, 상품유형, 금융기관, 원문 URL이 중요하다. 사용자는 보통 "청년도약계좌 가입 조건"처럼 상품명 중심으로 묻기 때문이다. 그래서 `institution`, `productName`, `productType`, `sourceUrl`을 추가했다.

그 다음 PDF 원본과 처리 결과의 위치를 나누었다. `data/raw`에는 원본 PDF를 두고, `data/processed`에는 chunk JSONL과 import report 같은 산출물을 두었다. 원본은 검증 기준이므로 Git에 포함할 수 있게 했고, 처리 결과는 언제든 다시 만들 수 있으므로 Git에서 제외했다.

PDF 추출은 `scripts/extract_busanbank_zip.py`와 `scripts/import_busanbank_installment_pdfs.py`로 나누었다. zip을 푸는 일과 PDF를 MySQL에 넣는 일을 분리한 것이다. importer는 PyMuPDF로 페이지 텍스트를 뽑고, 약 1,400자 단위로 chunk를 만든 뒤 `financial_documents`에 저장한다. PDF 전체를 하나로 넣지 않은 이유는 검색할 때 본문이 너무 길면 질문과 관련된 작은 구간이 묻히기 때문이다.

상품명 추론에서도 문제가 있었다. 파일명에는 날짜, 괄호, 개정 여부, 오타가 섞여 있었다. 특히 `정기적금`처럼 짧은 상품명이 `가계우대정기적금`보다 먼저 잡히면 잘못된 상품명으로 저장될 수 있었다. 그래서 alias와 상품명은 긴 문자열부터 매칭하도록 바꾸었다. 파일명에서 못 찾으면 PDF 앞쪽 텍스트를 보조로 사용했다.

2차에서는 프론트도 크게 줄였다. 사용자는 프론트를 이해하고 싶지 않다고 했고, 프로젝트의 핵심도 프론트가 아니었다. 그래서 문서 목록과 문서 등록 화면은 제거했다. 화면은 빈 메인 페이지와 우하단 플로팅 챗봇 버튼, 챗봇 패널만 남겼다. 실제 문서 데이터는 raw 폴더와 importer로 넣는다.

UI를 단순화하면서도 깨지는 부분은 고쳤다. `hidden` 속성이 CSS에 덮여 닫힌 챗봇이 보이거나, 입력바가 퀵버튼을 덮거나, 아바타가 원 안에 제대로 들어오지 않는 문제가 있었다. 이 문제는 기능보다 사용자 확인에 영향을 주기 때문에 최소한의 CSS 정리를 했다. 다만 프론트는 끝까지 핵심 학습 대상에서 제외했다.

2차에서 수정한 핵심 파일은 `FinancialDocument`, PDF importer 스크립트, 정적 프론트 파일 세 개, 그리고 Python RAG 검색 corpus였다. 이 단계의 결론은 "RAG 품질은 모델보다 데이터 형태에서 먼저 결정된다"는 것이었다.

## 2차에서 남긴 판단

PDF 자동 수집기는 만들지 않았다. 홈페이지 구조가 바뀔 수 있고 다운로드 링크가 동적일 수 있으며, 최신성 검증까지 들어가면 범위가 커지기 때문이다. 대신 현재 단계에서는 사용자가 제공한 PDF를 안정적으로 적재하고 검색하는 데 집중했다.

OCR도 바로 붙이지 않았다. 일부 표 안 텍스트가 이미지일 수 있지만, 먼저 PyMuPDF 텍스트 레이어만으로 어느 정도 검색되는지 확인하는 것이 낫다고 판단했다. OCR은 정확도, 속도, 설치 난이도 문제가 있으므로 실제로 빠진 정보가 확인되면 도입하는 것이 맞다.

## 용어 메모

- 메타데이터: 본문 외에 문서를 설명하는 부가 정보다. 상품명, 출처 URL, 상품유형 등이 여기에 해당한다.
- PyMuPDF: PDF에서 페이지별 텍스트를 추출하는 Python 라이브러리다.
- JSONL: 한 줄에 JSON 객체 하나씩 저장하는 파일 형식이다.
- importer: 외부 파일을 읽어 DB에 적재하는 스크립트나 프로그램이다.
- raw 데이터: 가공하지 않은 원본 데이터다.
- processed 데이터: 원본을 추출, 변환, 정리해 만든 가공 데이터다.
- OCR: 이미지 안의 글자를 텍스트로 바꾸는 기술이다.
