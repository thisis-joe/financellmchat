# 10. LoRA와 QLoRA 학습 데이터셋 설계

이 문서는 이 프로젝트에서 LoRA와 QLoRA를 어디에 쓰고, 어떤 데이터 포맷으로 준비해야 하는지 설명한다.  
핵심은 상품 조건을 모델에 외우게 만드는 것이 아니라, 검색된 근거를 읽고 안전한 형식으로 답변하는 습관을 학습시키는 것이다.

## 먼저 결론

이 프로젝트에서 LoRA/QLoRA의 목표는 다음이다.

```text
좋은 역할:
  - RAG 근거를 읽고 핵심만 요약하는 답변 형식 학습
  - 금융 상품공시 답변 말투 학습
  - 근거에 없는 내용은 말하지 않는 습관 학습
  - 출처와 추가 확인 필요 문구를 안정적으로 붙이는 습관 학습

나쁜 역할:
  - 특정 상품의 금리, 가입대상, 서류를 모델에 외우게 하기
  - 최신 공시 내용을 모델 파라미터에 저장하기
  - RAG 검색 없이 모델 기억만으로 답하게 만들기
```

금융 상품 조건은 바뀔 수 있다. 따라서 최신 사실은 항상 PDF 문서 검색으로 가져오고, LoRA/QLoRA는 답변 방식만 보정하는 데 쓴다.

## LoRA와 QLoRA 차이

LoRA는 큰 모델 전체를 다시 학습하지 않고 작은 adapter만 학습한다. 그래서 학습 비용이 낮고, 원본 모델을 보존한 채 프로젝트에 맞는 답변 습관을 얹을 수 있다.

QLoRA는 양자화된 모델 위에서 LoRA를 학습한다. 더 큰 모델을 적은 GPU 메모리로 다루기 위한 방식이다. 데이터 포맷은 LoRA와 거의 같다. 차이는 데이터가 아니라 학습 런타임과 메모리 전략에 있다.

```text
LoRA
  -> 원본 모델은 고정
  -> 작은 adapter만 학습
  -> 품질과 안정성 확인이 비교적 단순

QLoRA
  -> 원본 모델을 4bit 등으로 양자화
  -> 작은 adapter만 학습
  -> 더 큰 모델을 적은 메모리로 실험 가능
```

설명할 때는 이렇게 말하면 된다.

> 이 프로젝트는 상품 지식을 모델에 외우게 하지 않습니다. RAG가 문서를 찾아오고, LoRA/QLoRA는 그 근거를 읽어 `핵심 답변`, `확인 내용`, `출처`, `추가 확인 필요` 형식으로 안정적으로 답하는 방식을 학습시키는 역할입니다.

## 데이터 생성 출발점

학습 데이터의 출발점은 다음이다.

```text
query_histories
  -> 사용자가 실제로 물은 질문
  -> 모델 또는 fallback이 만든 답변
  -> citationsJson에 저장된 검색 근거

answer_feedbacks
  -> HELPFUL / NOT_HELPFUL 평가
  -> 좋은 답변 선별 기준
```

좋은 학습 데이터는 `HELPFUL`이 붙은 답변에서 시작한다. `NOT_HELPFUL` 답변은 그대로 학습시키지 않고, 사람이 고친 정답 답변을 만들어야 한다.

## 권장 JSONL 포맷

파일 경로는 다음처럼 둔다.

```text
data/training/lora/rag_answer_sft.jsonl
```

`data/training`은 생성 산출물이므로 Git에 꼭 넣을 필요는 없다. 문서에는 포맷을 남기고, 실제 학습 데이터는 필요할 때 생성한다.

한 줄은 하나의 학습 예시다.

```json
{
  "id": "bnk-rag-sft-000001",
  "task": "rag_answer_style",
  "messages": [
    {
      "role": "system",
      "content": "당신은 BNK부산은행 상품공시 PDF를 근거로 답하는 상담 보조 챗봇입니다. 근거에 없는 내용은 추측하지 말고, 필요한 경우 추가 확인 필요를 안내합니다."
    },
    {
      "role": "user",
      "content": "질문: 펫 적금 혜택 받으려면 뭘 해야 해?\n\n근거:\n[1] 문서: 펫 적금 상품설명서\n상품명: 펫 적금\n내용: 동물등록증 제출, 펫 다이어리 작성, 거래실적에 따른 우대이율..."
    },
    {
      "role": "assistant",
      "content": "핵심 답변: 펫 적금 혜택을 받으려면 상품설명서의 우대이율 조건을 확인해야 합니다.\n\n확인 내용:\n- 동물등록증 제출 여부를 확인합니다.\n- 펫 다이어리 작성 등 우대 조건을 확인합니다.\n- 거래실적에 따른 우대이율 적용 여부를 확인합니다.\n\n출처: 펫 적금 상품설명서\n\n추가 확인 필요: 실제 적용 여부는 가입 시점의 고시일과 은행 확인 결과에 따라 달라질 수 있습니다."
    }
  ],
  "metadata": {
    "domain": "BNK부산은행 상품공시",
    "category": "예금상품>적립식예금",
    "source_history_id": 123,
    "citations": [
      {
        "documentId": 1,
        "title": "펫 적금 상품설명서",
        "productName": "펫 적금",
        "score": 0.91
      }
    ],
    "feedback": "HELPFUL",
    "split": "train"
  }
}
```

중요한 필드는 `messages`다. 대부분의 SFT 도구는 chat messages 포맷을 바로 받거나 쉽게 변환할 수 있다.

## 필드 설명

`id`는 학습 예시를 추적하기 위한 값이다.

`task`는 이 데이터가 무엇을 학습시키는지 나타낸다. 현재는 `rag_answer_style`을 기본으로 둔다.

`messages`는 실제 모델이 학습할 대화다. `system`에는 역할과 안전 규칙을 넣고, `user`에는 질문과 근거 문서를 함께 넣으며, `assistant`에는 기대 답변을 넣는다.

`metadata`는 학습에는 직접 쓰지 않아도 된다. 하지만 나중에 어떤 history와 어떤 PDF 근거에서 나온 데이터인지 추적하기 위해 남긴다.

## 답변 품질 기준

학습 답변은 다음 기준을 지켜야 한다.

```text
1. 질문에 직접 답한다.
2. 검색 근거에 있는 내용만 사용한다.
3. 상품 조건은 단정하지 않고 가입 시점 확인을 안내한다.
4. 답변 구조는 핵심 답변, 확인 내용, 출처, 추가 확인 필요를 따른다.
5. 내부 chunk 번호나 p2-1 같은 디버그 이름은 사용자 답변에 넣지 않는다.
6. 출처는 파일 제목 중심으로 남긴다.
```

좋지 않은 데이터는 제외한다.

```text
제외:
  - 근거와 다른 상품을 추천한 답변
  - 청년 전용 상품을 50대에게 추천한 답변
  - 출처 없는 단정적 답변
  - 모델이 지시문을 복사한 답변
  - 문장이 중간에 끊긴 답변
```

## 데이터 분할

처음에는 작게 시작한다.

```text
train: 80%
validation: 10%
test: 10%
```

중요한 것은 질문 유형이 한쪽에 몰리지 않게 하는 것이다. 가입대상, 금리, 해지, 서류, 추천, 범위 밖 질문, 대화성 입력을 골고루 나눈다.

## LoRA 실험 순서

추천 순서는 다음이다.

```text
1. 대표 질문 20~30개를 먼저 안정화한다.
2. query_histories와 feedback을 모은다.
3. HELPFUL 답변을 후보로 뽑는다.
4. 사람이 좋은 답변 형태로 정리한다.
5. rag_answer_sft.jsonl을 만든다.
6. 작은 LoRA 실험으로 답변 형식이 좋아지는지 확인한다.
7. 필요하면 QLoRA로 더 큰 모델을 검토한다.
```

바로 QLoRA부터 가지 않는 이유는 문제를 나누기 위해서다. 먼저 데이터 포맷과 평가 기준이 맞는지 LoRA로 확인하고, 모델 크기나 메모리 한계가 문제가 될 때 QLoRA로 넓힌다.

## 평가 방법

학습 후에도 평가 기준은 RAG답게 가져간다.

```text
좋은 결과:
  - 답변 형식이 안정적이다.
  - 근거에 없는 내용을 덜 말한다.
  - 출처와 추가 확인 필요를 잘 붙인다.
  - 같은 대표 질문에서 답변이 덜 흔들린다.

나쁜 결과:
  - 상품 조건을 외운 것처럼 말한다.
  - 검색 근거가 부족해도 단정한다.
  - 출처를 꾸며낸다.
  - 현재 적재되지 않은 대출/카드 자료까지 답하려고 한다.
```

LoRA/QLoRA가 성공했는지는 말투가 좋아졌는지보다 대표 질문과 근거 일치율이 유지되는지로 봐야 한다.

## 용어 메모

- SFT: Supervised Fine-Tuning. 질문과 정답 답변 예시로 모델을 지도학습하는 방식이다.
- LoRA: 원본 모델은 고정하고 작은 adapter만 학습하는 파인튜닝 방식이다.
- QLoRA: 양자화된 모델 위에서 LoRA adapter를 학습해 메모리 사용량을 줄이는 방식이다.
- adapter: 원본 모델 옆에 붙는 작은 학습 모듈이다.
- JSONL: 한 줄에 JSON 객체 하나를 저장하는 데이터 파일 형식이다.
- messages 포맷: system, user, assistant 역할을 가진 대화형 학습 데이터 구조다.
- train/validation/test: 학습용, 검증용, 최종 평가용 데이터 분할이다.
- 근거 일치율: 답변 내용이 검색된 문서 근거와 얼마나 일치하는지 보는 기준이다.
